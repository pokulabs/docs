---
title: "Webhook"
description: "Set up a webhook so your OpenClaw agent can receive incoming texts and calls."
---

Poku needs a public URL to forward inbound SMS and call summaries to your OpenClaw agent. How you set this up depends on where your agent is running.

---

## Step 1: Get a public URL

### Option A — Local machine (via ngrok)

If OpenClaw is running on your local machine, use ngrok to create a public tunnel.

1. Create an account at [ngrok.com](https://ngrok.com).

2. After logging in, follow the installation instructions for your operating system. On macOS you can install via Homebrew:

   ```bash
   brew install ngrok
   ```

3. Grab your authtoken from the [ngrok dashboard](https://dashboard.ngrok.com/get-started/your-authtoken) and authenticate:

   ```bash
   ngrok config add-authtoken <your-token>
   ```

4. Start a tunnel pointed at OpenClaw's default port:

   ```bash
   ngrok http 18789
   ```

   ngrok will output a forwarding URL:

   ```
   Forwarding https://abc123.ngrok-free.app -> http://localhost:18789
   ```

   Copy this URL "https://abc123.ngrok-free.app" — you'll need it in Step 3.

---

### Option B — VPS (no ngrok needed)

If OpenClaw is hosted on a VPS, your agent is already on the public internet — no tunnel required.

**Recommended:** Set up HTTPS using a reverse proxy such as nginx or Caddy with a free Let's Encrypt certificate. This gives you a clean, secure URL:

   ```
   https://yourdomain.com
   ```

   And avoids exposing a raw port in your webhook URL.


If you are not using a reverse proxy, follow the below steps:
1. Navigate to your cloud provider (AWS, DigitalOcean, Hetzner, etc.) and open the port that your agent is running on (ex.18789) in your provider's security group or firewall dashboard.

2. Your webhook URL is your server's public IP or domain:

   ```
   http://YOUR_VPS_IP:18789
   ```

   Or if you have a domain pointed at your VPS:

   ```
   http://yourdomain.com:18789
   ```

## Step 2: Update your OpenClaw config

Add a Poku mapping to the `hooks` section of `~/.openclaw/openclaw.json`. For full details on OpenClaw webhook configuration, see the [OpenClaw webhook documentation](https://docs.openclaw.ai/automation/webhook).

We have provided an example config below.


```json
{
  "hooks": {
    "enabled": true,
    "token": "<openclaw-hooks-token>",
    "transformsDir": "~/.openclaw/hooks/transforms",
    "defaultSessionKey": "hook:ingress",
    "allowRequestSessionKey": false,
    "allowedSessionKeyPrefixes": [
      "hook:"
    ],
    "mappings": [
      {
        "match": {
          "path": "poku"
        },
        "action": "agent",
        "agentId": "main",
        "sessionKey": "hook:poku",
        "wakeMode": "now",
        "name": "Poku",
        "deliver": true,
        "channel": "telegram",
        "to": "<your-telegram-id>",
        "messageTemplate": "You received a message {{payload.payload.from}}: \"{{payload.payload.body}}\" {{payload.payload.humanResponse}}"
      }
    ]
  }
}
```

The Poku payload includes the following fields:

| Field | Description |
|-------|-------------|
| `payload.from` | Phone number the text or call was from |
| `payload.body` | Body of the incoming text |
| `payload.humanResponse` | Summary of the inbound call |

---

## Step 3: Register your webhook with Poku

Register your public URL from Step 1 with Poku so inbound texts get forwarded to your agent:

```bash
curl -X POST "https://api.pokulabs.com/webhooks" -H "Authorization: Bearer <your-poku-api-key>" -H "Content-Type: application/json" -d '{"url":"<your-webhook-url>","eventTypes":["phone.sms.received","phone.call.received"],"headers":{"Authorization":"Bearer <openclaw-hooks-token>"},"reservedPhoneNumberIds":["<your-reserved-phone-id"]}'
```

---

## Verify your webhook

You can check which webhook URL is currently registered on your account:

```bash
curl -X GET https://api.pokulabs.com/webhooks \
  -H 'Authorization: Bearer <your-poku-api-key>'
```

<Note>
  This endpoint returns a list of registered webhooks.
</Note>

---

## Delete your webhook

You can delete any webhook URL registered on your account:

```bash
curl -X DELETE "https://api.pokulabs.com/webhooks/<webhook-id>" -H "Authorization: Bearer <your-poku-api-key>"
```
