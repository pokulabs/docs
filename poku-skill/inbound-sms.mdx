---
title: "Inbound SMS"
description: "Set up a webhook so your OpenClaw agent can receive and respond to incoming texts."
---

Poku needs a public URL to forward inbound SMS to your OpenClaw agent. How you set this up depends on where your agent is running.

---

## Step 1: Get a public URL

### Option A — Local machine (via ngrok)

If OpenClaw is running on your local machine, use ngrok to create a public tunnel.

1. Create an account at [ngrok.com](https://ngrok.com).

2. After logging in, follow the installation instructions for your operating system. On macOS you can install via Homebrew:

   ```bash
   brew install ngrok
   ```

3. Grab your authtoken from the [ngrok dashboard](https://dashboard.ngrok.com/get-started/your-authtoken) and authenticate:

   ```bash
   ngrok config add-authtoken <your-token>
   ```

4. Start a tunnel pointed at OpenClaw's default port:

   ```bash
   ngrok http 18789
   ```

   ngrok will output a forwarding URL:

   ```
   Forwarding https://abc123.ngrok-free.app -> http://localhost:18789
   ```

   Copy this URL — you'll need it in Step 3.

---

### Option B — VPS (no ngrok needed)

If OpenClaw is hosted on a VPS, your agent is already on the public internet — no tunnel required.

1. Open port 18789 in your firewall. On Ubuntu with ufw:

   ```bash
   ufw allow 18789
   ```

   If you're using a cloud provider (AWS, DigitalOcean, Hetzner, etc.), also open the port in your provider's security group or firewall dashboard.

2. Your webhook URL is your server's public IP or domain:

   ```
   http://YOUR_VPS_IP:18789
   ```

   Or if you have a domain pointed at your VPS:

   ```
   http://yourdomain.com:18789
   ```

3. **Recommended:** Set up HTTPS using a reverse proxy such as nginx or Caddy with a free Let's Encrypt certificate. This gives you a clean, secure URL:

   ```
   https://yourdomain.com
   ```

   And avoids exposing a raw port in your webhook URL.

## Step 2: Update your OpenClaw config

Add the following `hooks` section to `~/.openclaw/openclaw.json`:

```json
{
  "hooks": {
    "enabled": true,
    "token": "SET_YOUR_OPENCLAW_HOOKS_TOKEN",
    "transformsDir": "~/.openclaw/hooks/transforms",
    "defaultSessionKey": "hook:ingress",
    "allowRequestSessionKey": false,
    "allowedSessionKeyPrefixes": [
      "hook:"
    ],
    "mappings": [
      {
        "match": {
          "path": "poku"
        },
        "action": "agent",
        "agentId": "main",
        "sessionKey": "hook:poku",
        "wakeMode": "now",
        "name": "Poku",
        "deliver": true,
        "channel": "<your-channel>",
        "to": "<your-channel-id>",
        "messageTemplate": "New inbound SMS from {{payload.payload.from}}: {{payload.payload.body}}"
      }
    ]
  }
}
```

Set `channel` to whichever notification channel you've configured in OpenClaw (e.g. `telegram`, `slack`), and `to` to the corresponding recipient ID for that channel.

---

## Step 3: Register your webhook with Poku

Register your public URL from Step 1 with Poku so inbound texts get forwarded to your agent:

```bash
curl -X POST https://api.pokulabs.com/webhooks \
  -H 'Authorization: Bearer $POKU_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://<your-public-url>/hooks/poku",
    "eventTypes": ["phone.sms.received"],
    "headers": {
      "Authorization": "Bearer OPENCLAW_HOOKS_TOKEN"
    }
  }'
```

---

## Verify your webhook

You can check which webhook URL is currently registered on your account:

```bash
curl -X GET https://api.pokulabs.com/webhooks \
  -H 'Authorization: Bearer $POKU_API_KEY'
```

<Note>
  This endpoint returns only the most recently registered webhook.
</Note>
